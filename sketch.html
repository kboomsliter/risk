RISK

After playing RISK this Christmas with the kids I thought it'd be fun to explore making a web-based version as a coding challenge. Considering I barely know Python but want to learn, and knowing nothing about web UI, this could be a great learning experience.

Game design

Every action in the game is public with all players. It seems that server side will be needed to commit most actions. Troop placement can be worked out client side but a commit/submit action will make it so on the server side. Dice roll is all server once the attacker and defender commit the number of troops to battle.

Game Initialize

  • Number of players - determines numbers of troops
  • Determine play order
    • Roll one die, highest goes first
    • Determine how to order remaining players (more die rolls to break ties?)
    • Assign troops color
  • Game mode
    • Classic RISK (build this first, probably)
    • Secret Mission
    • RISK for 2 Players
    • Capital RISK

Game Setup

  • Classic RISK
    • 3 players - 35 Infantry each
    • 4 players - 30 Infantry each
    • 5 players - 25 Infantry each
  • Players claim territories with 1 Infantry in turn until all territories are claimed

Classes

    • Occupied Troop instance (gets color)
